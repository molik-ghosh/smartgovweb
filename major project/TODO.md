## Comment Style Guide Applied

- **File Headers**: Multi-line block comments with file name, purpose, and section index
- **Section Dividers**: Visual ASCII separators (e.g., `// ── Section Name ──`)
- **JSDoc**: Function descriptions, `@param` types, and `@returns` where applicable
- **Inline Comments**: Brief explanations for complex logic or non-obvious code
- **Variable Comments**: Descriptive comments for state variables and constants
